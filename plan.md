
  # Предложение улучшений TaskFlow

  ## Обязательные функции, которых не хватает

  1. [x] **Единый экран “Сегодня / Мой день”**
     - Собирает в одном месте: недельный план на сегодня, назначенные задачи, оперативные задачи, просрочки, блокировки.
     - Для руководителя: “кому сегодня нужно внимание”.
     - Для исполнителя: “что конкретно сделать сейчас”.

  2. **Напоминания и эскалации**
     - Напоминать о дедлайнах, просрочках, задачах без движения.
     - Эскалация ответственному/админу, если задача зависла.
     - Использовать уже существующие уведомления и управленческий контроль как основу.

  3. **Повторяющиеся задачи и шаблоны**
     - Ежедневные/еженедельные задачи для личного плана и оперативки.
     - Шаблоны задач/эпиков: “создать типовой набор подзадач”.
     - Сильно сократит ручной ввод.

  4. **Вложения и ссылки в задачах**
     - Фото, документы, ссылки, сканы, файлы подтверждения.
     - Сейчас в схеме видны комментарии/активность, но не видно полноценной модели вложений.

  5. **Нормальный глобальный поиск**
     - Поиск по задачам, эпикам, комментариям, людям, личному плану.
     - Command Palette уже есть, но её стоит расширить до настоящего рабочего поиска.

  6. **Админский экран резервного копирования и восстановления**
     - В README описаны операции с БД/backup, но для обычного администратора нужен UI: скачать backup, загрузить
  backup, проверить здоровье БД.
     - Это обязательная функция для локальной интранет-системы.

  ## Новые функции, которые упростят жизнь

  1. **Очередь управленческого внимания 2.0**
     - Не просто список рисков, а действия: “назначить ответственного”, “перенести срок”, “написать комментарий”,
  “отметить причину блокировки”.
     - Быстрые кнопки прямо из `/management`.

  2. **Календарь по людям и подразделениям**
     - Неделя/месяц: задачи, недельный план, дедлайны, оперативные события.
     - Фильтр по пользователю, роли, составу.
     - Экспорт в CSV/ICS.

  3. **Причины блокировки**
     - Для статуса `blocked` требовать короткую причину.
     - Показывать блокировки отдельным блоком руководителю.
     - Это сделает статус полезным, а не просто “красной колонкой”.

  4. **Чек-листы/шаблонные подзадачи**
     - Быстро добавлять набор подзадач: “проверить”, “согласовать”, “доложить”, “закрыть”.
     - Особенно полезно для повторяемых процессов.

  5. **Права и роли по действиям**
     - Сейчас роли и permissions уже заложены, но стоит довести до понятной матрицы: кто может создавать, назначать,
  закрывать, редактировать, экспортировать.
     - В UI настроек показать это как таблицу прав.

  6. **Журнал изменений прямо в карточке задачи**
     - Кто поменял срок, статус, ответственного, описание.
     - Таблица `task_activity` уже есть — её нужно сделать видимой и полезной.

  7. **Импорт задач из Excel/CSV**

  8. **Быстрые команды из палитры**
     - “Создать задачу”, “назначить на меня”, “показать мои просрочки”, “открыть сегодняшнее”.
     - Сейчас палитра больше навигационная; её можно превратить в центр действий.

  1. **Починить mojibake в документации и части комментариев**
     - В текущем worktree README и часть комментариев/строк отображаются как битая кириллица.
     - Для эксплуатации это критично: инструкции деплоя и администрирования должны быть читаемыми.

  2. **Свести статусы/метаданные к единому источнику**
     - В проекте уже есть `shared/config/task-meta.ts`, но статусы повторяются в разных UI.
     - Особенно важно после auth/access изменений.
  3. **После:** backup/restore UI + календарь + импорт CSV/Excel.
  4. **Параллельно:** починка mojibake и унификация статусов.
